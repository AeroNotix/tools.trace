API for clojure.tools.trace - Execution tracing tool 0.7.3 (in development)

by Stuart Sierra, Michel Salim, Luc Préfontaine, Jonathan Fischer Friberg, Michał Marczyk, Don Jackson
Full namespace name: clojure.tools.trace

Overview

Project home page is http://github.com/clojure/tools.trace/
This file defines simple tracing macros to help you see what your code is doing.

Public Variables and Functions


*trace-depth*

var

    
Current stack depth of traced function calls.

deftrace

    macro

    
Usage: (deftrace name & definition)


    
Use in place of defn; traces each call/return of this fn, including
arguments. Nested calls to deftrace'd functions will print a
tree-like structure.
The first argument of the form definition can be a doc string

    
    
    Source
  

    

    

    
dotrace

    macro

    
Usage: (dotrace fnames & exprs)


    
Given a sequence of function identifiers, evaluate the body
expressions in an environment in which the identifiers are bound to
the traced functions. Does not work on inlined functions,
such as clojure.core/+

    
    
    Source

trace

    function

    
Usage: (trace value)
       (trace name value)


    
Sends name (optional) and value to the tracer function, then
returns value. May be wrapped around any expression without
affecting the result.

    
    
    Source
  

    

    

    
trace-compose-exception

    function

    
Usage: (trace-compose-exception exception message)


    
Re-create a new exception with a composed message from the given exception
and the message to be added. The exception stack trace is kept at a minimum.

    
    
    Source
  

    

    

    
trace-fn-call

    function

    
Usage: (trace-fn-call name f args)


    
Traces a single call to a function f with args. 'name' is the
symbol name of the function.

trace-ns*

    function

    
Usage: (trace-ns* ns)


    
Replaces each function from the given namespace with a version wrapped
in a tracing call. Can be undone with untrace-ns. ns should be a namespace
object or a symbol.

No-op for clojure.core and clojure.tools.trace.

    
    
    Source
  

    

    

    
trace-var*

    function

    
Usage: (trace-var* ns s)
       (trace-var* v)


    
If the specified Var holds an IFn and is not marked as a macro, its
contents is replaced with a version wrapped in a tracing call;
otherwise nothing happens. Can be undone with untrace-var.

In the unary case, v should be a Var object or a symbol to be
resolved in the current namespace.

In the binary case, ns should be a namespace object or a symbol
naming a namespace and s a symbol to be resolved in that namespace.

    
    
    Source
  

    

    

    
trace-vars

    macro

    
Usage: (trace-vars & vs)


    
Macro to wrap calls to trace-var* on each of the specified Vars.
The vs may be Var objects or symbols to be resolved in the current
namespace.

    
    
    Source
  

    

    

    
tracer

    function

    
Usage: (tracer name value)


    
This function is called by trace. Prints to standard output, but
may be rebound to do anything you like. 'name' is optional.

untrace-ns*

    function

    
Usage: (untrace-ns* ns)


    
Reverses the effect of trace-var / trace-vars / trace-ns for the
Vars in the given namespace, replacing each traced function from the
given namespace with the original, untraced version.

    
    
    Source
  

    

    

    
untrace-var*

    function

    
Usage: (untrace-var* ns s)
       (untrace-var* v)


    
Reverses the effect of trace-var / trace-vars / trace-ns for the
given Var, replacing the traced function with the original, untraced
version. No-op for non-traced Vars.

Argument types are the same as those for trace-var.

    
    
    Source
  

    

    

    
untrace-vars

    macro

    
Usage: (untrace-vars & vs)


    
Macro to wrap calls to untrace-var* on each of the specified Vars.
Reverses the effect of trace-var / trace-vars / trace-ns for each
of the vs, replacing the traced functions with the original,
untraced versions.
